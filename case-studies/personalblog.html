Personal Blog Case Study

Context

The summer after my freshman year, I was a Lead Software Instructor at iD Tech Summer Camps at Emory University where I taught kids aged 7-17 how to code using Python, Lua scripting, and Adobe Photoshop. However, during my off days, I still wanted to learn new technologies that I didn't have the chance to work with after my first-year at Brown. Therefore, I made it my goal that summer to create a personal project that I could present on my resume while allowing me to explore new technologies that I haven't had the chance to use before.

End-of-the-week Celebration at iD Tech Emory

Part 1: Research

During my off days from work, there were a lot of different technologies that I was interested in.


  • First, I was interested in potentially doing a project in sports analytics involving machine learning . As someone who has played soccer his whole life, I loved the idea of creating a project that would be an intersection between my interests in sports and computer science. However, after some thought, I thought this wouldn't be the best idea for this personal project during the summer. The reason is because I had to consider the audience that I wanted to appeal to. Not only did I want to make something that was interesting, but my primary audience that I wanted to appeal to was recruiters as recruiters would be the first people that would lay eyes on my resume.
  • Therefore, I wanted to make a web application, so that recruiters would be able to view the application with their own eyes to break down the technical barrier. Then, I began to ask myself, what should I make my theme for my application? Additionally, at the time, I had just finished reading James Clear's Atomic Habits and was a big fan of his personal blog. Therefore, I made it my goal that summer to create a personal blog using technologies that I haven't used before while working 60+ hours a week as an instructor. Not only would this give me an avenue to write articles about topics that I enjoyed, but I would be exploring new technologies during the process.
  • Technologies: Next, I needed to figure out what technologies to use for my personal blog. At the time (and probably still), I heard of this Javascript framework called React.js that was very popular.
    Therefore, I primarily wanted to use React.js. Additionally, as someone who wanted to try out new styling packages, I decided on Tailwind CSS as my primary styling language. After searching online on what would be the best ways to store data, I decided on using GraphQL, Next.js, and Hygraph. Therefore, the technologies that I wanted to use would be the following:
    • React.js
    • Next.js
    • GraphQL
    • Hygraph
    • Tailwind CSS
  • With the technologies and the idea in mind, I started to begin the design process.

Part 2: Design Iterations

Initial Design Sketches


I had a strong vision for the design of the blog before implementing the code. I've included the sketches for my initial designs below:

Main Homepage

Specific Post

As shown in my sketches above, I wanted the main layout of the my blog to include blog posts on the left hand side with a recent posts and categories section on the right hand side for filtering of posts. Therefore, as you vertically scroll down, you can see previous posts that were made. Then, if you are interested in specific categories or the most recent posts, then you can see those options on the side.



However, I made some additional considerations after these initial sketches:

  • One additional consideration was providing some sort of security whenever a user makes a comment. Therefore, I thought it would be cool to have a system that whenever a user enters a comment, myself or an admin could approve that comment in the database, and only then will it be shown on the application.
    • Tradeoff: The tradeoff here was that if I implemented this additional layer of security, then the user wouldn't see their comment being posted right away, which is not as fun. However, I thought that the positive of additional security would outweigh this instant form of gratification to make sure that nobody could just comment anything on the website.
  • Another consideration was incorporating a carousel system at the top of the application in additional to the scroll-down-posts layout. I thought this could be cool to have the "most important posts" at the top of the screen: kind of like a news feed.
    • Tradeoff: The potential negative here is that it could lead to clutter at the top of the screen; however, I thought it would be important to have a "most important" posts section analogous to a lot of media outlets, so I ultimately made the design to keep this design choice as well.

Implementation Challenges


One of my favorite parts was actually implmenting the design that I laid out for myself. However, here were some challenges that I ran into while implementing:

  • First, the hardest part of the project for me was querying through GraphQL. It was a language that I've never really used before, and it had a steep learning curve. Therefore, I had to spend a lot of time on YouTube and on Stack Overflow working on bugs and trying to find efficient ways to retrieve information from my Hygraph database.
  • Next, it was challenging learning all of the syntax for Tailwind CSS, and oftentimes my main React components would get extremely cluttered due to the amount of classnames there were. However, I thought it was a refreshing way to do CSS, and eventually I liked the way that it was ordered per component element rather than just nested in one large CSS file.
  • Finally, while React.js and Next.js was initially difficult to get used to eventually it started to become one of my favorite languages to use. Fastforward a couple of years, React.js is probably one of my favorite languages to code applications in, and I've used the language in both of my previous internships.

Part 4: Learnings

There was so much that I learned from completing this project. First, I was able to learn new technologies that I haven't had the chance to use before such as Next.js and GraphQL. Second, I was able to create a personal blog which is something that I was inspired to do based off James Clear's own personal blog. Third, I was able to publish this application on my resume after the summer of the my freshman year, and as a junior in college, it is still on my resume as one of my proudest projects.

Coincidentally, in the next semester after I returned to college, I received an email from a local startup looking for a software engineer intern. Interested, as I looked at the qualifications for the role, I was surprised to find that the "expected technologies" included React.js, Next.js, and Tailwind CSS which were all technologies that I used for this project. During my interview process, the developer mainly asked questions about my personal blog: the inspiration behind it as well as the design intuition that led to the creation of the application. To my surprise, we spent most of the thirty minute interview just talking about my blog, and then a week later, I got the acceptance email for a software engineer intern position at Siege.



As a result, this project is one of my most favorite projects. Still in interviews to this day, I still talk about how this project with recruiters and developers alike. The main takeaway that I've received is that in order to create a personal project, having a strong passion and idea before starting the implementation is key to staying motivated as you never know what benefit a single project could have to your portfolio down the line.
